Bootnode

Concept

Bootnode is a simple way to load javascript on the fly. If you have a feature that is not critical to the functioning of the page, you may prefer to load this dependency later rather than including it with the page. Using data attributes directly in the HTML code, Bootnode allows you to declaratively associate a dependency with some user action, deferring its download until this action has been detected.

Activation

In the tetra configuration, set the enableBootnode parameter to true. This will activate bootnode for all applications. 

tetra.extend({conf:{}})
Definition in markup

Attributes for configuring bootnode must be set on a top-level or containing element of your application interface. There are 3 parameters to set

class="bootnode" : adding 'bootnode' to the element class permits Tetra to find which element is reactive to bootnode actions.

data-event="click" : the event that will trigger the download of the associated dependency. This attribute supports the click, mouseover and focus event type.

data-view="appName/viewName" : the path (scope/name) to the view of the associated dependency. This view, along with its own dependencies (controller and models), will be downloaded by Tetra when the data-event action is invoked.

data-comp="compName" : To reduce HTTP requests, we can optionally associate the data-event with a packaged version of our dependency. Replace data-view with data-comp, providing a custom name compName for the packaged JavaScript.

<div class="bootnode" data-view="likeApp/likeButton" data-event="click">
...
</div>
<div class="bootnode" data-comp="likePackage" data-event="click">
...
</div>

Dependency management system

A view depends on one or more controllers, specified in its "use" parameter. Similarly, a controller depends on one or more models. When a bootnode event is invoked, Tetra downloads the view and its associated controllers and models. It then initializes each part of the application in the following order: models then controllers then view. Once finished, the data-event action is invoked.
